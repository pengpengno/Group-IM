# Group-IM 项目环境配置指南

本文档全面梳理了 Group-IM 项目（包括后端服务层与跨平台前端）所需的开发与运行环境配置。

## 1. 后端环境配置 (@server)

后端采用 **Java 21** + **Spring Boot** 的现代技术栈，主要依赖以下中间件与服务：

### 1.1 核心依赖与版本要求
*   **JDK版本**: Java 21 (推荐使用 GraalVM 或标准 OpenJDK 21)
*   **构建工具**: Maven (项目内含 `mvnw` wrapper，无需全局安装)
*   **Spring Boot**: 3.x (带有 Webflux, Spring Security, Data JPA, Data Redis)
*   **数据库**: PostgreSQL `42.7.3` (JDBC驱动)
*   **缓存**: Redis (通过 Lettuce `6.5.0.RELEASE` 连接)
*   **AI能力**: Spring AI (支持 OpenAI / Groq 兼容接口)
*   **实时通信**: 核心使用 WebSocket 与 WebRTC
*   **其他组件**: LDAP（轻量级目录访问协议用户认证），Alibaba Transmittable-Thread-Local

### 1.2 中间件与基础设施准备
推荐使用 Docker 或 Docker Compose 在本地启动相关中间件：
1.  **PostgreSQL**:
    *   **默认端口**: `5432`
    *   **默认账号/密码**: `postgres` / `postgres`
    *   **数据库名**: `group`
2.  **Redis**:
    *   **默认端口**: `6379`
    *   **不需要默认密码** (可通过 `${REDIS_HOST}` 调整)
3.  **LDAP** (可选身份认证):
    *   **默认连接**: `ldap://localhost:389`
    *   **管理员DN**: `cn=admin,dc=mycompany,dc=com` (密码: `admin`)
4.  **WebRTC STUN/TURN**:
    *   默认 STUN: `stun:stun.l.google.com:19302`
    *   默认 TURN: `turn:localhost:3478` (本地开发阶段可关闭或配置自定义)

### 1.3 启动后端项目配置
可以在本地IDE (IntelliJ IDEA) 中配置启动参数，也可修改 `server/src/main/resources/application-local.yml`：
```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
  datasource:
    url: jdbc:postgresql://localhost:5432/group
    username: postgres
    password: postgres
```
本地运行命令：
```sh
cd server
../mvnw spring-boot:run -Dspring-boot.run.profiles=local
```

## 2. 前端环境配置 (@Group)

前端采用 **Kotlin Multiplatform (KMP)** 与 **Compose Multiplatform** 构建，实现 Android, Desktop, iOS 及 Web 的代码复用。

### 2.1 核心依赖与版本要求
*   **Kotlin 版本**: `2.1.21`
*   **AGP (Android Gradle Plugin)**: `8.7.3`
*   **Compose Multiplatform**: `1.8.1`
*   **UI 框架**: Jetpack Compose / Material 3 (`1.2.1`)
*   **依赖注入**: Koin `4.0.4`
*   **网络请求**: Ktor `3.1.3`
*   **本地数据库**: SQLDelight `2.0.1`
*   **路由/导航**: Voyager `1.1.0-beta02`
*   **异步机制**: Kotlin Coroutines `1.10.2`
*   **序列化**: Kotlinx Serialization `1.9.0`, Wire `4.8.1`, Protobuf `4.30.0`
*   **JDK版本**: 建议使用 Java 17 编译 Android 和 KMP 代码

### 2.2 开发工具与环境
*   **推荐IDE**: Android Studio (Koala 或更新版本) 或 IntelliJ IDEA Ultimate / Fleet。
*   **插件要求**: 安装或更新 `Kotlin Multiplatform Mobile` (KMM) 插件和 `Compose Multiplatform IDE Support` 插件。
*   **SDK要求**: 需安装 Android SDK (minSdk `24`, targetSdk `35`)。如需开发 iOS 版本，需配置 macOS 环境并安装 Xcode 工具链。

### 2.3 启动前端客户端
前端客户端代码位于 `Group-app/Group`（构建脚本使用 Gradle Kotlin DSL）。

1.  **Android 客户端**:
    ```bash
    ./gradlew :Group-app:Group:composeApp:installDebug
    ```
2.  **Desktop 桌面端 (Windows/macOS/Linux)**:
    ```bash
    ./gradlew :Group-app:Group:composeApp:run
    ```

## 3. 常见问题排查 (Troubleshooting)

1.  **Redis 连接失败**: 检查 Spring Boot 3.5.x 与 Lettuce 的兼容性问题，当前已通过锁定 Lettuce 核心版本为 `6.5.0.RELEASE` 解决，请确保依赖正确拉取。
2.  **Gradle 同步失败 (前端)**: 请确保当前使用的 Java 版本与 Gradle 要求的版本匹配 (通常推荐 Java 17 给 Gradle daemon)。
3.  **数据库连接超时**: 检查本地 PostgreSQL 服务是否已启动并已创建名为 `group` 的数据库实例。可以参照 `deploy` 或 `docker` 目录下的 compose 文件快速起服务。
