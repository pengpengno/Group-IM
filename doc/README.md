## 🗂 文档分类说明

### 功能文档 (Functional Documentation)
描述系统功能、使用方法和业务流程
- 面向：产品经理、测试人员、最终用户
- 内容：功能说明、操作指南、业务流程

### 功能设计 (Design Documentation)
系统设计、架构设计、详细设计文档
- 面向：开发人员、架构师
- 内容：架构图、时序图、设计方案

### 部署运维 (Deployment & Operations)
部署指南、运维手册、监控告警等
- 面向：运维人员、DevOps 工程师
- 内容：部署流程、配置说明、故障排查

### 单元测试 (Testing Documentation)
测试规范、测试用例、测试报告
- 面向：测试人员、开发人员
- 内容：测试计划、测试用例、质量报告

### 音视频 (Media Documentation)
WebRTC、音视频通话相关技术文档
- 面向：音视频开发工程师
- 内容：WebRTC 实现、媒体处理

## 📋 文档规范

### 命名规范
- 中文文档：`中文名称.md`
- 英文文档：`kebab-case-name.md`
- UML 图：`图表名称.puml`

### 结构规范
遵循统一的文档模板:
1. 概述
2. 主体内容
3. 实践步骤
4. 常见问题
5. 参考资料
6. 更新日志

详见：[文档模板](./文档模板.md)

### 质量要求
- ✅ 准确性：内容经过验证
- ✅ 完整性：包含必要信息
- ✅ 可读性：逻辑清晰、图文并茂
- ✅ 时效性：及时更新维护

## 🔄 文档更新

### 更新原则
- 新功能开发同步更新文档
- Bug 修复后更新相关说明
- 定期审查文档时效性
- 过时内容标注或删除

### 贡献流程
```
1. Fork 项目
2. 创建文档分支 (feature/doc-xxx)
3. 编写/修改文档
4. 提交变更
5. 发起 Pull Request
6. 团队审核合并
```

### 版本控制
- 重大更新：主版本号升级 (v1.0 → v2.0)
- 小幅修改：次版本号升级 (v1.0 → v1.1)
- 在文档末尾记录变更历史
