# 人类可读的文档编写技能指南

本文档是 `documentation-writer.yaml` 的人类可读版本，详细说明了如何编写高质量的技术文档。

## 📖 核心能力

### 1. 文档结构设计
遵循以下原则设计文档目录：
- **MECE 原则**: 相互独立，完全穷尽
- **用户导向**: 从读者角度出发
- **渐进式**: 由浅入深，由宏观到微观

**推荐结构**:
```
doc/
├── README.md              # 总导航
├── 部署运维/
│   ├── README.md
│   ├── CI-CD 标准流程.md
│   └── 阿里云部署指南.md
├── 开发指南/
└── 功能文档/
```

### 2. 内容创建
标准文档结构包含：
1. **概述** (100-300 字)
2. **目标** (通过阅读本文能够...)
3. **前置条件** (需要了解的知识)
4. **主体内容** (分章节详细说明)
5. **实践步骤** (可操作的步骤)
6. **常见问题** (FAQ)
7. **最佳实践**
8. **故障排查**
9. **参考资料**
10. **更新日志**

### 3. 格式标准化

#### 文件命名
- 中文文档：`描述性名称.md`
- 英文文档：`kebab-case-name.md`

#### 标题层级
最多 4 级标题 (`#` 到 `####`)

#### 代码块
````markdown
```bash
# 示例命令
command here
```
````

### 4. 质量保证

#### 准确性 ✅
- 所有命令经过实际验证
- 代码示例可运行
- 版本号明确标注

#### 完整性 ✅
- 包含完整操作流程
- 提供故障处理方案
- 列出相关资源链接

#### 可读性 ✅
- 逻辑清晰、层次分明
- 图文并茂
- 术语统一

## 🛠️ 使用场景

### 场景 1: 创建 CI/CD 部署文档

**需求**: 项目使用 GitHub + 阿里云 + 宝塔，需要配置自动化部署

**输出文档**:
1. `doc/部署运维/CI-CD 标准流程.md`
2. `doc/部署运维/阿里云部署指南.md`
3. `doc/部署运维/宝塔面板配置手册.md`
4. `doc/部署运维/README.md` (导航索引)

**关键点**:
- 提供完整的 GitHub Actions 配置文件
- 包含 Docker Compose 配置详解
- 详细说明服务器配置步骤
- 提供监控告警和回滚方案

### 场景 2: 整理文档目录结构

**需求**: 项目文档杂乱无章，需要重新组织

**步骤**:
1. 分析现有文档内容和关系
2. 设计分类体系（按类型或模块）
3. 创建导航索引（README.md）
4. 重命名和移动文档到正确位置
5. 建立文档间的引用关系

**输出结构**:
```
doc/
├── README.md (总导航)
├── 部署运维/
│   ├── README.md
│   ├── CI-CD 标准流程.md
│   └── 阿里云部署指南.md
├── 开发指南/
└── 功能文档/
```

## 📋 检查清单

### 文档完成前检查
- [ ] 标题准确反映内容
- [ ] 结构符合规范
- [ ] 无错别字和语法错误
- [ ] 代码示例已验证
- [ ] 图表清晰可读
- [ ] 链接有效
- [ ] 格式统一

### 文档质量评估
- [ ] 是否包含所有必要章节？
- [ ] 是否有完整的操作步骤？
- [ ] 是否提供了示例代码？
- [ ] 是否有故障处理指南？
- [ ] 所有命令是否可执行？
- [ ] 代码示例是否可运行？
- [ ] 结构是否清晰？
- [ ] 语言是否简洁？

## 🎯 最佳实践

1. **先设计结构，再填充内容**
2. **使用统一的文档模板**
3. **为不同类型的文档提供变体**
4. **建立文档间的导航关联**
5. **定期审查和更新文档**
6. **收集读者反馈持续改进**
